# Wireframes da vitrine CORTEX

## Opção A, dor primeiro

Hero abre com a frustração do agente genérico. A primeira dobra mostra o contraste entre pedir
contexto toda vez e operar com memória externa. Boa para público ainda consciente só do problema.

## Opção B, mecanismo central

Hero coloca o mecanismo no centro: pedido solto entra, CORTEX roteia camadas, resposta operacional
sai. Boa para produto premium, porque vende o sistema completo. Esta é a opção aplicada.

## Opção C, compatibilidade primeiro

Hero abre com os runtimes suportados e deixa o mecanismo como prova abaixo. Boa para página de
documentação ou venda para público técnico que já sabe que precisa de contexto estruturado.
