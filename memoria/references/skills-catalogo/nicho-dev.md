# Skills pro nicho: desenvolvimento

Use junto com o pacote geral (geral.md).

Skills e categorias que mais rendem pra quem desenvolve software, mantém sistemas ou entrega
código como produto.

---

## Skills recomendadas

| Skill / Categoria | Pra quê | Como obter |
|-------------------|---------|------------|
| `pdf` (oficial) | Ler especificações técnicas em PDF, extrair requisitos, gerar documentação de API em PDF fechado para o cliente. | Via `/plugin` (proprietária Anthropic) |
| `xlsx` (oficial) | Criar planilhas de acompanhamento de bugs, matrices de teste, roadmap com estimativas. Menos frequente que em operações, mas útil em reuniões de escopo e entregas. | Via `/plugin` (proprietária Anthropic) |
| Revisão de código | Analisar um diff ou arquivo, apontar problemas de qualidade, segurança e simplificação, e sugerir (ou aplicar) melhorias cirúrgicas. Sem alterar o que não foi pedido. | Criar com `skill-creator` |
| Geração de testes | Ler uma função ou módulo e escrever testes unitários ou de integração no framework do projeto (Jest, Pytest, Go test, etc.). | Criar com `skill-creator` |
| Documentação de código | A partir de funções/módulos, gerar docstrings, comentários e um README estruturado. Salva o tempo de documentar manualmente. | Criar com `skill-creator` |
| Teste de webapp (headless) | Abrir o projeto no navegador, navegar nos fluxos críticos, tirar screenshot e reportar o que quebrou. Útil antes de fazer PR ou deploy. | Criar com `skill-creator` ou buscar no marketplace |
